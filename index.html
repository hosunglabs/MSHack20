HosungNotes X Jarvis Architecture

Users build knowledge on HosungNotes based on Azure. HosungNotes made using the ASP.NET MVC5. The knowledge builds up in MSSQL and downloads it into Word. After downloading, use the word file to train Azure Bot Jarvis. The training is based on Azure AI Service, QnAMaker.ai.

Therefore, users can always call in from the website of HosungNotes and from the Teams bot Jarvis to acquire knowledge.

Next

What is Hosung-Notes?

It is a web program where you can save your knowledge and find it easily.

First, there is a private ID, Password, in HosungNotes. If you don't have an ID or password, you can never share or find knowledge.

Second, accurate search is possible. Find the Index of all DataBase and show the information.

Third, you can put tags, titles, and content in knowledge. Tags are for anyone to easily find and study, and the title and content will be the content of the Case task. In addition, content can be copied and pasted easily, and any typical Editor function is available.

What is the Jarvis bot?

Jarvis is an Azure Bot program that works with Teams.

Jarvis has added Azure QnAMaker.ai, an Azure Cognitive Service. This made it easy for Jarvis to learn from the knowledge in HosungNotes.

In addition, Jarvis includes lunch or get-together locations, monthly events, company-required URLs, and club information.

Next

Meet HosungNotes' knowledge in the Microsoft Teams.

You can easily find it in Teams using Jarvis, who has learned HosungNotes through Teams. You can also call Jarvis directly on the Teams channel to check and discuss Jarvis' answers with the team members.

Next

Jarvis is using Azure Cognitive Service, QnAMaker.ai.

Click the brain icon and attach the knowledge received in Wordfiles from HosungNotes to Azure Cognitive Service QnAMaker.ai then AI will complete questions and answers automatically. Therefore, Jarvis can be learned quickly and easily and can be used.
